# KubeMin-Agent 可加强需求表单（2026-03-25）

## 1. 项目现状快照

- 代码主链路稳定：`pytest -q` 通过（`245 passed`）。
- 质量检查现状：`ruff check .` 通过，`mypy kubemin_agent` 未通过（45 个错误）。
- 覆盖率现状：总覆盖率约 `76%`，低覆盖集中在 `channels/`、`bus/`、`cli/ui.py`、`mcp_client.py`、`browser.py`。
- 文档与实现存在不一致：部分文档仍描述 legacy 链路，和当前“仅 ControlPlaneRuntime”现状冲突。

## 2. 关键可优化点（按影响排序）

1. 类型系统未闭环：CI 仅跑 `ruff + pytest`，缺失 `mypy` 质量门禁，且当前已有 45 个类型错误。
2. 配置项未真正生效：`max_tool_iterations` 在配置里可见，但 `BaseAgent` 仍硬编码 `MAX_ITERATIONS = 20`。
3. 警告噪声偏高：测试仍有 `RuntimeWarning`（AsyncMock 未 await）与 Pydantic V2 deprecation。
4. 文档“真相源”有偏差：部分文档继续描述 `AgentLoop`/`agent/context.py`，与当前代码状态不一致。
5. 消息总线扩展性不足：inbound/outbound 队列无上限，出站分发为单协程串行回调，存在慢通道阻塞风险。
6. 会话与审计存储策略偏基础：同步文件写入、全量缓存，缺少归档/轮转/保留策略。
7. 调度执行存在重复 I/O：`PlanExecutor` 在循环中多次读取 active plan 文件。
8. Shell 工具“工作区限制”语义不够强：`cwd` 切换并不等于真实文件系统隔离（需依赖 sandbox 才成立）。
